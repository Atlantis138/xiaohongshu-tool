# 小红书工具箱需求路线图

本项目的定位是本地运行的小红书脚本工具箱，而不是单一关键词抓取脚本。所有功能都应保持低频、人工可接管、可断点续跑，并遵守平台规则。

## 当前核心能力

- 关键词笔记采集：输入关键词和目标数量，采集搜索结果笔记详情。
- 搜索链接采集：只收集搜索结果中的笔记 URL，保存为 CSV。
- 链接 CSV 续跑：从已有链接 CSV 逐条访问详情页，适合长任务拆分。
- 三种采集策略：
  - `url_pipeline`：先收集 URL，再逐个打开详情页。
  - `preview_overlay`：在搜索页点击卡片预览层采集详情。
  - `new_tab`：在搜索页用新标签页打开卡片采集详情。
- 基础运行保障：登录状态复用、断点续跑、全局去重、随机延迟、定期休息、错误暂停人工接管。

## 近期可扩展需求

- 按笔记 URL 批量补全：导入任意笔记 URL 列表，补全标题、正文、作者、互动数等字段。
- 用户主页笔记采集：输入用户主页 URL，低频采集公开笔记列表和详情。
- 评论采集：在笔记详情基础上采集可见评论，单独输出评论 CSV。
- 数据清洗工具：对已有 CSV 做去重、字段标准化、失败行筛选、按关键词拆分。
- 关键词任务模板：保存常用关键词批量任务，GUI 可直接加载和复用。

## 后续增强方向

- 定时监控：以低频任务监控关键词或用户主页的新笔记，并输出增量结果。
- 导出格式扩展：在 CSV 外增加 JSON、Excel 导出。
- 质量报告：统计成功率、失败原因、空字段比例、重复跳过数量。
- 失败重试队列：将失败行输出为单独 CSV，便于下一次只重试失败任务。
- 任务配置文件：用 YAML/JSON 保存完整任务配置，CLI 和 GUI 共用。

## 合规边界

- 不做验证码绕过、滑块绕过、风控绕过。
- 不做接口签名逆向、私有 API 调用、请求伪造。
- 不做代理池、账号池、高并发或批量规避限制。
- 检测到登录、验证、访问异常、操作频繁等检查点时，默认停止或等待人工处理。
- 默认保持低频采集、显式延迟、可中断和可恢复。
